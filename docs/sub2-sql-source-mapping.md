# sub2 SQL Source Mapping

This note defines the minimal Azure SQL source layer now in use in `sub2` for the live Fabric mirroring path.

## Scope

The current star schema notebook reads exactly seven transactional source tables from the mirrored Azure SQL source in `sqldemo-mirror`.

| Source table | Generated by | Published by | Consumed by | Notes |
|---|---|---|---|
| `products` | `nb_01_setup_demo_environment.py` | `nb_05a_publish_synthetic_data_to_sql` | `nb_03_pbi_star_schema` via mirror | Product master / rate-card source |
| `customers` | `nb_01_setup_demo_environment.py` | `nb_05a_publish_synthetic_data_to_sql` | `nb_03_pbi_star_schema` via mirror | Customer master |
| `service_accounts` | `nb_01_setup_demo_environment.py` | `nb_05a_publish_synthetic_data_to_sql` | `nb_03_pbi_star_schema` via mirror | Service/account grain bridge |
| `equipment_registry` | `nb_01_setup_demo_environment.py` | `nb_05a_publish_synthetic_data_to_sql` | `nb_03_pbi_star_schema` via mirror | Installed equipment source |
| `contracts` | `nb_01_setup_demo_environment.py` | `nb_05a_publish_synthetic_data_to_sql` | `nb_03_pbi_star_schema` via mirror | Contract / subscription source |
| `service_requests` | `nb_01_setup_demo_environment.py` | `nb_05a_publish_synthetic_data_to_sql` | `nb_03_pbi_star_schema` via mirror | Service event source |
| `billing_transactions` | `nb_01_setup_demo_environment.py` | `nb_05a_publish_synthetic_data_to_sql` | `nb_03_pbi_star_schema` via mirror | Billing fact source |

## Initial load order

Load the tables into Azure SQL in the following order to satisfy foreign-key dependencies:

1. `products`
2. `customers`
3. `service_accounts`
4. `equipment_registry`
5. `contracts`
6. `service_requests`
7. `billing_transactions`
